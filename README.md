## Описание
В данном проекте представлена нейросеть, способная распознавать рукописные цифны. Обучение и тестирование проводилось на общедоступном датасете MNIST, содержащем 60000 цифр в обучающей выборке и 10000 цифр в тестовой. Все цифры в датасете представлены в двух файлах: пиксельный и файл с ответами. В пиксельном файле цифра хранится в виде матрицы пикселей 28х28, где каждый элемент матрицы - яркость пикселя от 0 до 1 (0 - чёрный, 1 - белый) (чёрно-белый формат). А также цифра хранится в файле с ответами в виде обычного числа.

Во время обучения, нейросеть загружает пиксельный файл и пытается для каждой матрицы предсказать, что за цифра на ней изображена. А далее, сравнивая предсказание с правильным ответом во втором файле, как-то меняет свои параметры.

В основе обучения лежит стандартный алгоритм Градиентного спуска. Более подробно детали реализации можно посмотреть в самих файлах.
## Установка
Чтобы склонировать репозиторий, вам понадобится Git. Далее прописываете следующие команды

```bash
# Clone this repository
$ git clone git@github.com:Doughnutik/Neural_Network_HSE.git

# Go into the repository
$ cd Neural_Network_HSE
```

Проверьте, что у вас установлен сmake. Если нет, то его можно легко установить через командную строку.

```bash
# Ubuntu installation
$ sudo apt install cmake

# Mac OS installation
$ brew install cmake
```

Далее нужно собрать проект. Для этого выполняем следующие шаги

```bash
$ mkdir build && cd build

$ cmake ..

$ cmake --build .
```

Если на вашей системе чего-то не хватает, необходимого для компоновки, cmake выдаст ошибку. Более подробно, что требуется, можно посмотреть в файле **CMakeLists.txt**, а также в логах cmake.
В случае успешной сборки, в папке build появятся два исполняемых файла **network** и **convert**.

## Как использовать
Исполняемый файл **convert** нужен, чтобы преобразовывать закодированные данные MNIST в читаемый и удобный для считывания txt формат.
```bash
# Usage

$ ./convert [option1] <path-to-file1> [option2] <path-to-file2> ...

# Options

-i <path-to-file>      = Конвертировать пиксельный файл
-l <path-to-file>      = Конвертировать файл с ответами
```
В итоге в папке с исходными файлами появятся файлы с такими же названиями, только с добавочным расширением .txt. Замечу, что в репозитории уже есть датасет MNIST в одноимённой папке. Достаточно просто запустить **convert** с этими файлами.

1) Исполняемый файл **network** и есть нейросеть. В ней уже реализованы удобные вопросы, которые помогут всё запустить. Отмечу лишь некоторые моменты.
2) Есть 2 режима работы - обучение и тестирование (**train**, **test**). В зависимости от введённого режима, нейросеть либо будет обучаться, либо будет предсказывать.
3) В режиме **train** для начало надо будет выбрать, конфигурировать ли нейросеть заново или загрузить веса из файла (опции **c**, **d**). В режиме тест можно только загрузить веса из файла.
4) Далее нужно указать файл, откуда будет загружаться конфиг или веса (в зависимости от выбранного параметра **c**, **d**). Для всех файлов можно указывать как абсолютный путь, так и относительный.
5) После этого надо ввести пути до txt файлов с пикселями и ответами.

Опишу, как задаётся файл конфигурации сети:\
**num_layers** **learning_rate**\
**l_1 l_2 l_3 ... l_num_layers**\
**function**\
\
Где **num_layers** - количество слоёв, **learning_rate** - double число, указывающее плавность\крутость изменения весов (см в коде)\
**l_1, ..., l_num_layers** - кол-ва нейронов на каждом слое соответственно\
**function** - название функции активации (пока что есть 3: sigmoida, relu, th). Быстрее всех работает relu, т.к. требует простых вычислений.
